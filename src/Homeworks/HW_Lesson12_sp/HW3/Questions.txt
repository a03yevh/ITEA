· Розкажіть про XML, у чому різниця між HTML?
· Які парсери в XML знаєте?
· Що робить JSON?
· Що таке JAXB?


Серіалізація/десеріалізація: Серіалізація - це процес перетворення структури даних або об'єкта в послідовність бітів, щоб зберегти його стан (наприклад, у файлі або в пам'яті),
або щоб передати його через мережу. Десеріалізація - це процес перетворення цієї послідовності бітів назад в первісну структуру даних або об'єкт.

Маршаллінг/демаршаллінг: Маршаллінг - це процес перетворення структури даних або об'єкта в формат, що може бути збережений або переданий, і він подібний до серіалізації.
Можна сказати, що маршаллінг - це більш спеціалізована форма серіалізації, яка використовується для передачі об'єктів між різними частинами системи (наприклад, між процесами
або між машинами через мережу). Демаршаллінг - це процес перетворення збереженого або переданого формату назад в структуру даних або об'єкт.

XML і різниця між HTML: XML (eXtensible Markup Language) - це мова розмітки, яку використовують для зберігання і передачі даних між системами. Вона використовує теги
для обозначення структури даних. HTML (HyperText Markup Language) - це інша мова розмітки, яка використовується для створення веб-сторінок. Вона використовує теги
для визначення структури та вигляду веб-сторінки. Головна різниця між ними полягає в тому, що HTML визначає, як дані виглядають, тоді як XML визначає, що ці дані означають.

Парсери XML: Існують різні парсери XML, серед яких DOM (Document Object Model) Parser, SAX (Simple API for XML) Parser, StAX (Streaming API for XML) Parser і JAXB
(Java Architecture for XML Binding) Parser.

Що робить JSON: JSON (JavaScript Object Notation) - це формат обміну даними, що використовується для зберігання і передачі даних між браузером і сервером.
JSON використовується для серіалізації даних в простий та зручний для читання формат. JSON став популярним альтернативою XML для передачі даних через мережу, особливо
в контексті веб-застосунків.

JAXB: JAXB (Java Architecture for XML Binding) - це API в Java, який використовується для маршаллінгу (перетворення) Java об'єктів у XML і демаршаллінгу (перетворення)
XML назад в Java об'єкти. JAXB дозволяє Java розробникам оперувати XML даними без необхідності знати XML або XML парсинг.
